# ADR 0001: Repository Structure and Multi-Solution Strategy

## Status
Accepted

## Context
پروژه Zamin طی زمان به صورت یک framework چندبخشی رشد کرده است و اکنون توسط چند تیم توسعه استفاده می‌شود. ساختار مخزن شامل چند solution مستقل است و برخی بخش‌ها مانند Onion، Extensions و Utilities به صورت جداگانه نگهداری می‌شوند.

همچنین در ساختار فعلی دو ناحیه با مفهوم Utilities وجود دارد:

1. پوشه `Utilities` در ریشه مخزن که شامل پروژه‌های زیرساختی، registrationها و capabilityهای مستقل است.
2. پوشه `Onion/src/1.Utilities` که شامل utilityهای پایه مرتبط با معماری Onion است.

با توجه به تعدد مصرف‌کنندگان و اهمیت پایداری پکیج‌ها، تغییرات ساختاری باید کم‌ریسک، تدریجی و قابل مستندسازی باشند.

## Decision
- solutionها فعلاً مستقل باقی می‌مانند و به یک solution واحد ادغام نمی‌شوند.
- مدیریت متمرکز build، package management، versioning و مستندسازی در سطح repository انجام خواهد شد.
- تفاوت بین `Utilities` در ریشه و `Onion/src/1.Utilities` باید در مستندات رسمی به‌وضوح ثبت شود.
- هرگونه merge، rename یا حذف package در آینده فقط همراه با مستند migration انجام خواهد شد.
- تصمیم‌های معماری مهم از این به بعد در ADRها ثبت می‌شوند.

## Consequences
- مرزهای مفهومی solutionها حفظ می‌شود.
- مدیریت پروژه در سطح repository نیازمند استانداردسازی بیشتر خواهد بود.
- مستندسازی و migration guide نقش مهم‌تری در توسعه و نگهداری خواهند داشت.
- بازآرایی‌های آینده باید تدریجی و با حفظ تجربه مصرف‌کنندگان انجام شوند.
