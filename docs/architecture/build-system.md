## ساختار نهایی build

در وضعیت فعلی، فقط یک `Directory.Build.props` و یک `Directory.Packages.props` در ریشه repository نگه داشته می‌شود.

این تصمیم با هدف‌های زیر گرفته شد:

- حذف پراکندگی تنظیمات
- ساده‌سازی نگهداری build
- مدیریت متمرکز dependencyها
- کاهش ریسک ناسازگاری بین solutionهای مختلف

در این مدل، همه پروژه‌های repository تنظیمات build و نسخه dependencyهای خود را از ریشه دریافت می‌کنند و فقط تنظیمات خاص هر پروژه در `csproj` آن باقی می‌ماند.
