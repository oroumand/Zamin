# نمای کلی Zamin

پروژه Zamin یک framework چندبخشی برای توسعه نرم‌افزار در اکوسیستم .NET است که در ابتدا برای استفاده داخلی توسعه داده شد و به مرور توسط چند تیم به عنوان ابزار توسعه مورد استفاده قرار گرفت.

این framework بر پایه تفکیک مسئولیت‌ها، لایه‌بندی شفاف و امکان استفاده مجدد از اجزای مشترک شکل گرفته است. هسته اصلی آن بر مبنای معماری Onion توسعه داده شده و در کنار آن مجموعه‌ای از extensionها و utilityها برای سناریوهای مختلف فراهم شده است.

## بخش‌های اصلی

### Onion
بخش مرکزی framework که ساختار و اجزای اصلی معماری Onion را فراهم می‌کند. این قسمت شامل لایه‌های دامنه، application services، قراردادها، request/responseها، زیرساخت داده و endpointهای وب است.

### Extensions
مجموعه‌ای از abstractionها و adapterها برای قابلیت‌های عمومی مانند object mapping، serialization، caching، events، message bus، dependency injection، database integration و موارد مشابه.

### Utilities
این بخش در دو ناحیه متفاوت وجود دارد:

- `Utilities` در ریشه مخزن: شامل پروژه‌های زیرساختی، registrationها و capabilityهای مستقل.
- `Onion/src/1.Utilities`: شامل utilityهای پایه مرتبط با معماری Onion مانند تاریخ، زمان، تبدیل انواع و ابزارهای پایه.

## هدف مستندات
این مستندات با هدف‌های زیر تهیه می‌شوند:

- ساده‌سازی onboarding تیم‌های جدید
- شفاف‌سازی ساختار و مسئولیت هر package
- مستندسازی تصمیم‌های معماری
- پشتیبانی از مهاجرت به .NET 10
- فراهم کردن راهنمای توسعه و نگهداری framework
